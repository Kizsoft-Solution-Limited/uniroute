# API Keys Explained: UniRoute vs AI Providers

## üîë Understanding the Two Types of API Keys

When using UniRoute, there are **two completely different types of API keys**:

---

## 1. UniRoute API Keys (Your Access to UniRoute)

### What They Are
- **Your authentication key** to use UniRoute's API
- **Format**: `ur_abc123def456...` (starts with `ur_`)
- **Who provides**: **UniRoute** (you get this from UniRoute)

### How to Get One
- **Self-hosted**: Generated by UniRoute when you create an API key
- **Hosted service**: Get from UniRoute dashboard after signup

### Where to Use
```javascript
// In your application code
const response = await fetch('https://api.uniroute.dev/v1/chat', {
  headers: {
    'Authorization': 'Bearer ur_your-uniroute-key'  // ‚Üê This is your UniRoute key
  },
  body: JSON.stringify({
    model: 'gpt-4',
    messages: [...]
  })
});
```

### Purpose
- Authenticate your requests to UniRoute
- Track your usage
- Apply rate limits
- Manage access control

### Cost
- **FREE** - UniRoute API keys are free to create
- **Usage charges apply** - You pay UniRoute for AI model usage (ONLY when using managed service)
- **FREE with BYOK** - No charges when you provide your own provider keys
- **FREE when self-hosting** - No charges when self-hosting UniRoute
- No cost to create or manage UniRoute API keys

---

## 2. AI Provider API Keys (Your Access to AI Models)

### What They Are
- **API keys from AI providers** (OpenAI, Anthropic, Google)
- **Format**: 
  - OpenAI: `sk-...`
  - Anthropic: `sk-ant-...`
  - Google: `AIza...`
- **Who provides**: **YOU** (you get these from the providers directly)

### How to Get Them

#### OpenAI
1. Go to https://platform.openai.com/api-keys
2. Sign up or log in
3. Create a new API key
4. Copy the key (starts with `sk-`)

#### Anthropic
1. Go to https://console.anthropic.com/
2. Sign up or log in
3. Navigate to API Keys
4. Create a new API key
5. Copy the key (starts with `sk-ant-`)

#### Google
1. Go to https://makersuite.google.com/app/apikey
2. Sign in with Google account
3. Create a new API key
4. Copy the key (starts with `AIza`)

### Where to Configure

#### Self-Hosted UniRoute
```bash
# In your .env file or environment variables
OPENAI_API_KEY=sk-your-openai-key-here
ANTHROPIC_API_KEY=sk-ant-your-anthropic-key-here
GOOGLE_API_KEY=AIza-your-google-key-here
```

#### Hosted UniRoute (if available)
- Configure in your UniRoute dashboard
- Settings ‚Üí Provider API Keys
- Enter your keys securely

### Purpose
- UniRoute uses these keys to make requests to AI providers
- **On your behalf** - UniRoute acts as a proxy
- You're still the one paying the providers

### Cost (BYOK Model)
- **You pay the providers directly**
- OpenAI bills you for OpenAI usage
- Anthropic bills you for Anthropic usage
- Google bills you for Google usage
- **UniRoute doesn't charge for using your provider keys**

### Cost (Managed Service Model)
- **UniRoute bills you** (includes provider costs)
- UniRoute pays the providers on your behalf
- Unified billing through UniRoute
- Pay-as-you-go pricing

## üîÑ How It Works Together

### The Flow:

```
Your Application
      ‚Üì
   [Uses UniRoute API Key: ur_abc123...]
      ‚Üì
   UniRoute Gateway
      ‚Üì
   [Uses Provider API Keys: sk-..., sk-ant-..., AIza...]
      ‚Üì
   AI Providers (OpenAI, Anthropic, Google)
      ‚Üì
   Response back through UniRoute
      ‚Üì
   Your Application
```

### Example Request Flow:

1. **Your app** sends request with **UniRoute API key**:
   ```javascript
   POST /v1/chat
   Authorization: Bearer ur_your-uniroute-key
   ```

2. **UniRoute** validates your UniRoute API key
   - Checks if key is valid
   - Applies rate limits
   - Logs the request

3. **UniRoute** uses **your provider API keys** to call AI providers:
   - Uses `OPENAI_API_KEY` to call OpenAI
   - Uses `ANTHROPIC_API_KEY` to call Anthropic
   - Uses `GOOGLE_API_KEY` to call Google

4. **AI providers** bill **YOU directly** (not UniRoute)

5. **UniRoute** returns the response to your app

## üéØ Common Questions

### Q: Do I need to pay UniRoute?
**A**: 
- **Managed Service**: Yes, you pay UniRoute for AI model usage (UniRoute pays providers on your behalf)
- **BYOK (Bring Your Own Keys)**: No, UniRoute does NOT charge you - you pay providers directly
- **Self-Hosted**: No, UniRoute is completely free - no charges at all

### Q: Can I use UniRoute without provider API keys?
**A**: Yes! You can use UniRoute with only local models (Ollama, vLLM) which require no API keys and are completely free.

### Q: What if I don't have provider API keys?
**A**: 
- You can still use UniRoute with local models
- Or get free API keys from providers (they often have free tiers)
- Or use UniRoute's hosted service (if available) which may provide keys

### Q: Are my provider API keys secure?
**A**: Yes! When self-hosting:
- Keys are stored in environment variables (never in code)
- Keys are encrypted at rest
- Keys are only used server-side (never exposed to clients)

### Q: Can I use multiple provider keys?
**A**: Yes! You can configure multiple keys for the same provider (useful for load balancing or different accounts).

### Q: What happens if I don't configure provider keys?
**A**: UniRoute will only use local models (Ollama, vLLM). Cloud providers won't be available until you configure their keys.
